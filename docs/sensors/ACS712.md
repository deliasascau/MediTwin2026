# ACS712 — Hall-Effect Current Sensor

## Ce este

Senzor de curent bazat pe efect Hall. Măsoară curentul AC sau DC care trece prin conductorul intern, fără contact galvanic cu circuitul măsurat.

Folosit în MediTwin pentru **fault detection al ventilatorului**:
- Ventilator comandat ON → ACS712 trebuie să vadă curent
- Dacă curent ≈ 0 → ventilator defect sau circuit întrerupt → **FAULT**

Aceasta este funcționalitatea "wow" a proiectului.

---

## Variante de modul

| Variantă | Sensibilitate | Range curent | Utilizare |
|---|---|---|---|
| ACS712-5A | 185 mV/A | ±5A | Motoare mici, ventilatoare |
| ACS712-20A | 100 mV/A | ±20A | Motoare medii |
| ACS712-30A | 66 mV/A | ±30A | Aplicații industriale |

Pentru ventilator mic de 5V/12V → **ACS712-5A** este potrivit.

## Pini modul

| Pin | Funcție | Note |
|---|---|---|
| VCC | Alimentare senzor | 5V |
| GND | Masă | |
| OUT | Ieșire analogică | ~1.65V la curent zero (3.3V supply) |
| IP+ | Intrare conductor | Curentul măsurat intră pe IP+ |
| IP- | Ieșire conductor | Curentul măsurat iese prin IP- |

## Schema circuitului

```
┌────────────────────────────────────────────────┐
│              CIRCUITUL DE PUTERE               │
│  5V ──── [ACS712 IP+] ──── [IP-] ──── Ventilator ──── GND
└────────────────────────────────────────────────┘

┌────────────────────────────────────────────────┐
│              CIRCUITUL DE SEMNAL               │
│  ESP32-C6 5V ──── ACS712 VCC                   │
│  GND          ──── ACS712 GND                  │
│  GPIO2 (ADC) ──── ACS712 OUT                   │
└────────────────────────────────────────────────┘
```

Curentul ventilatorului trece prin ACS712 (IP+ → IP-).  
Senzorul generează pe OUT o tensiune proporțională cu curentul.

## Montare pe breadboard — pas cu pas

**Important**: ACS712 are două circuite separate:
1. Circuitul de putere (IP+ / IP-)  — curentul ventilatorului
2. Circuitul de semnal (VCC/GND/OUT) — conectat la ESP32

```
CIRCUITUL DE PUTERE:
  5V sursa ──── ACS712 IP+
  ACS712 IP- ──── fir la ventilator VCC (+)
  Ventilator GND ──── GND comun

CIRCUITUL DE SEMNAL:
  ACS712 VCC ──── 5V ESP32 sau sursă
  ACS712 GND ──── GND
  ACS712 OUT ──── GPIO2 (ADC1_CH2)
```

> ⚠️ **OUT iese la tensiune proporțională cu VCC senzorului.**  
> Dacă ACS712 VCC = 5V, OUT poate depăși 3.3V la curenți mari.  
> Adaugă **rezistor 10kΩ în serie + 10kΩ pull-down** pe OUT (sau alimentează ACS712 VCC direct la 3.3V dacă modulul permite).

**Soluție sigură (ACS712 VCC la 3.3V):**
```
ESP32 3V3 ──── ACS712 VCC   (AOUT va fi maxim 3.3V → sigur pentru GPIO)
GND       ──── ACS712 GND
GPIO2     ──── ACS712 OUT   (direct, fără divizor)
```

## Cum funcționează calculul de curent

```
V_out = VCC/2 + (curent × sensibilitate)
```

La zero curent:
```
V_out = 3.3V / 2 = 1.65V  →  ADC = 4095 × (1.65/3.3) = 2047 ≈ 2048
```

La curent pozitiv (ex: 1A cu ACS712-5A):
```
V_out = 1.65V + (1A × 0.185V/A) = 1.835V
ADC   = 4095 × (1.835/3.3) ≈ 2278
```

Formula în cod:
```cpp
float avg     = suma_20_samples / 20;
float voltage = (avg / 4095.0) × 3.3;
float midV    = (2048 / 4095.0) × 3.3;
float current = (voltage - midV) / (185.0 / 1000.0);  // 185mV/A → 0.185V/A
```

## Verificare funcționare

Uploadează `meditwin_test`, trimite `7`.

**Faza 1 — calibrare zero:**
```
Zero-current raw ADC : 2051
Zero-current voltage : 1.6527 V
Configured MIDPOINT  : 2048
OK — midpoint within tolerance.
```

Dacă `abs(tău - 2048) > 100` → actualizează `ACS712_MIDPOINT` în `config.h`.

**Faza 2 — live current:**
```
Current: +0.001 A  [idle / no load]    ← fără ventilator
Current: +0.312 A  [low load]           ← ventilator pornit
Current: -0.002 A  [idle / no load]    ← normal: mică fluctuație spre negativ
```

## Calibrare — procedura completă

### Pasul 1 — Zero offset
Cu **niciun curent** prin IP+/IP- (ventilator deconectat):

1. Trimite `7` → notează `Zero-current raw ADC` → ex: `2063`
2. Actualizează în `config.h`:
```c
#define ACS712_MIDPOINT  2063   // valoarea ta reală
```

### Pasul 2 — Verificare cu sarcină
Conectează ventilatorul, pornește-l (`F` în test mode):
- Trebuie să vezi curent > 0.05A
- Dacă curentul rămâne la 0 cu ventilatorul evident pornit → problema e în circuit (IP+ / IP-)

### Pasul 3 — Prag fault detection
Curentul ventilatorului tău la mers normal. Notează valoarea.  
Setează pragul de fault la ~30% din acea valoare:

```c
// config.h
#define THRESH_FAN_CURRENT_MIN  0.05f  // A — sub asta = fault dacă fan e ON
```

## Reducere zgomot (important!)

ACS712 e sensibil la zgomot ADC. Codul face deja **medie pe 20 de sample-uri**.  
Dacă tot ai fluctuații mari, adaugă condensator:

```
ACS712 OUT ──── [100nF] ──── GND
```
Montat cât mai aproape de pinul OUT al modulului.

## Fault Detection Logic (în main.cpp)

```
fan_command = ON
ACS712 curent < 0.05A
→ stare = FAULT
→ LED galben blink
→ buzzer ON
→ Raspberry primește {"state": "FAULT", "reason": "fan_no_current"}
```

## Probleme frecvente

| Simptom | Cauză probabilă | Fix |
|---|---|---|
| ADC mereu ~2048, curentul mereu 0 | IP+/IP- nu sunt în serie cu sarcina | Pune ventilatorul **prin** IP+→IP- |
| Valori enorm de zgomotoase | Lipsă condensator de decuplaj | Adaugă 100nF la OUT-GND |
| Zero offset mare (> 2200) | VCC ACS712 e 5V nu 3.3V | Actualizează `ACS712_MIDPOINT` sau schimbă VCC |
| Curent negativ cu sarcina ON | IP+ și IP- inversate | Inversează firele sau acceptă valoarea absolută |
| ESP32 se resetează | OUT la 5V direct cu VCC=5V | Alimentează ACS712 la 3V3 sau adaugă divizor |

## Note pentru proiect MediTwin

ACS712 este **fault detection** — cel mai impresionant feature din perspectiva juriului:

> "Sistemul nu doar pornește ventilatorul. El **verifică** dacă ventilatorul chiar funcționează. Dacă pornește comanda dar nu vede curent, trece automat în FAULT și alertează."

**Script demo:**
1. Ventilator pornit normal → curent ~0.3A → sistem OK
2. Deconectezi manual firul ventilatorului → comanda e ON dar curent = 0 → **FAULT**
3. Dashboard: roșu + "Ventilation Failure Detected"

Pin definit în `config.h`: `PIN_ACS712 = GPIO2`  
Sensibilitate: `ACS712_MV_PER_A = 185.0` (pentru 5A)  
Midpoint: `ACS712_MIDPOINT = 2048` (actualizează după calibrare)
