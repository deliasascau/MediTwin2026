# MQ-3 — Air Quality Sensor

## Ce este

Senzor electrochimic de gaze care detectează:
- NH3 (amoniac)
- NOx (oxizi de azot)
- CO2
- Alcool, fum, benzen

Folosit în MediTwin ca **indicator principal de calitate a aerului**.  
Modulul are atât ieșire analogică (AOUT) cât și digitală (DOUT — cu comparator fix, nefolosit).

---

## Pini modul MQ-3

| Pin modul | Funcție |
|---|---|
| VCC | 5V (heaterul intern necesită 5V!) |
| GND | GND |
| AOUT | Ieșire analogică 0–5V (reducată la 3.3V prin divizor sau direct dacă modulul are regulator) |
| DOUT | Ieșire digitală (comparator fix — nu o folosim) |

> ⚠️ **VCC trebuie să fie 5V**, nu 3.3V. Heaterul intern al senzorului necesită 5V pentru a funcționa corect. Fără 5V, valorile sunt incorecte.

> ⚠️ **AOUT poate ieși la 5V**. GPIO-urile ESP32-C6 sunt **tolerante doar la 3.3V**. Verifică dacă modulul tău are un regulator de tensiune pe AOUT — dacă nu, adaugă divizor de tensiune.

---

## Schema circuitului

### Dacă modulul tău are regulator pe AOUT (ieșire 3.3V max):
```
ESP32-C6                MQ-3 modul
  5V  ─────────────────  VCC
  GND ─────────────────  GND
  GPIO1 (ADC) ──────────  AOUT
```

### Dacă AOUT poate depăși 3.3V (fără regulator — **mai sigur**):
```
MQ-3 AOUT ──── [10kΩ] ──── GPIO1
                     |
                  [20kΩ]
                     |
                    GND
```
Divizor 10kΩ / 20kΩ reduce 5V → 3.3V:  
`V_out = 5V × 20/(10+20) = 3.33V`

## Montare pe breadboard — pas cu pas

```
Componentă         Breadboard    Destinație
─────────────────────────────────────────────
MQ-3 VCC    →   rând 5 col A  →  fir la 5V (ESP32 sau sursă externă)
MQ-3 GND    →   rând 5 col B  →  fir la GND
MQ-3 AOUT   →   rând 5 col C  →  fir la GPIO1 (sau prin divizor)
```

**Cu divizor de tensiune (recomandat):**
```
MQ-3 AOUT
    │
  [10kΩ]  ← rândul 10
    │
  ──┬────── fir la GPIO1 (ESP32)
  [20kΩ]  ← rândul 13
    │
   GND
```

## Conexiuni ESP32-C6

| Pin MQ-3 | GPIO ESP32-C6 |
|---|---|
| VCC | **5V** (pin VBUS sau sursă externă) |
| GND | **GND** |
| AOUT | **GPIO 1** (ADC1_CH1) — prin divizor dacă e nevoie |
| DOUT | neconectat |

## Warm-up — IMPORTANT

MQ-3 are un **heater intern** care trebuie să atingă temperatura de operare.

| Timp de la pornire | Stare sensor |
|---|---|
| 0 – 30 s | Complet instabil — ignoră valorile |
| 30 s – 2 min | Parțial stabil — valori mari, scad |
| 2 – 5 min | Aproape stabil |
| 5+ min | **Stabil — citiri de încredere** |

Codul are `MQ135_WARMUP_MS = 10000` ms (10 s) si praguri dinamice pe baseline la fiecare boot.

**Pentru hackathon**: lasă senzorul pornit 5–10 minute înainte de demo.

## Calibrare

### Pasul 1 — Determină baseline-ul tău

Cu senzorul cald (5+ min), rulează testul de stabilitate (`0`).  
Noteaza valoarea **avg** a MQ-3 in aer curat. Ex: `avg = 1250`.

### Pasul 2 — Seteaza pragurile in config.h

```c
// Praguri dinamice curente (baseline + offset)
#define MQ135_OFFSET_WARNING   500
#define MQ135_OFFSET_CRITICAL  900
```

Regula curenta in proiect:
- `WARNING  = baseline + 500`
- `CRITICAL = baseline + 900`

---

## Interpretare valori

| Valoare ADC | Semnificație |
|---|---|
| < 1500 | Aer curat (în cameră normală) |
| 1500 – 1800 | Ușor viciat — ventilație recomandată |
| 1800 – 2500 | Calitate slabă — WARNING |
| > 2500 | Pericol — CRITICAL |

> Valorile exacte depind de mediu și de exemplarul senzorului! Calibrează întotdeauna.

---

## Probleme frecvente

| Simptom | Cauză probabilă | Fix |
|---|---|---|
| Valori mereu mari (3000+) | VCC = 3.3V în loc de 5V | Mută VCC la 5V |
| Valori nu se stabilizează | Sensor rece, warm-up insuficient | Așteaptă 5 min |
| Valori instabile chiar și la cald | Conexiuni slabe sau noise ADC | Verifică cablaj; adaugă 100nF la AOUT–GND |
| ADC mereu 0 | AOUT neconectat sau GPIO gresit | Verifica firul si `PIN_MQ135` |
| ESP32 se resetează | AOUT la 5V direct pe GPIO 3.3V | Adaugă divizor de tensiune |

## Note pentru proiect MediTwin

MQ-3 este **senzorul vedeta** al proiectului:
- Valoarea lui declanșează stările WARNING și CRITICAL
- Este input principal în modelul Random Forest
- Demo-ul se bazează pe el: "crești aer → galben → roșu"

**Pentru demo**: ține un marker permanent sau respiră adânc pe senzor pentru a crește rapid valoarea.

Pin definit in `config.h`: `PIN_MQ135 = GPIO1` (denumire legacy in cod)
