# MPU6050 — 6-Axis IMU (Accelerometer + Gyroscope)

## Ce este

Senzor MEMS cu 3 axe de accelerometru și 3 axe de giroscop, interfață I2C.  
Produs de InvenSense (acum TDK).

Folosit în MediTwin pentru:
- **Detecție vibrații** — tamper, impact, mișcare bruscă
- **Detecție tamper** — cineva mișcă/lovește echipamentul
- **Contribuție AI** — schimbări bruște de accelerație → factor de risc

---

## Pini

| Pin modul | Funcție | Note |
|---|---|---|
| VCC | Alimentare | 3.3V sau 5V (modulul are regulator) |
| GND | Masă | |
| SCL | I2C clock | |
| SDA | I2C data | |
| XDA | I2C auxiliar | Nefolosit — lasă liber |
| XCL | I2C auxiliar clock | Nefolosit — lasă liber |
| AD0 | Address select | GND = 0x68 / VCC = 0x69 |
| INT | Interrupt | Opțional — nefolosit în build curent |

---

## Adresa I2C

| AD0 (pin) | Adresa I2C |
|---|---|
| Legat la **GND** | **0x68** (default) |
| Legat la **3V3** | **0x69** |

Dacă ai BMP280 (0x76) și MPU6050 (0x68) pe același bus — adresele sunt diferite, funcționează simultan fără problemă.

---

## Schema circuitului

```
ESP32-C6          MPU6050 modul
  3V3  ──────────  VCC
  GND  ──────────  GND
  GPIO8 (SDA) ───  SDA
  GPIO9 (SCL) ───  SCL
  GND  ──────────  AD0   (adresa 0x68)
                   INT   — neconectat
                   XDA   — neconectat
                   XCL   — neconectat
```

> Dacă ai deja BMP280 pe GPIO8/GPIO9, MPU6050 se **conectează în paralel** pe același bus I2C — nu ai nevoie de fire separate.

## Montare pe breadboard — pas cu pas

**Dacă BMP280 e deja montat pe bus I2C:**

```
Componentă        Breadboard     Destinație
────────────────────────────────────────────────────
MPU6050 VCC    →  rând 20 A  →  fir 3V3
MPU6050 GND    →  rând 20 B  →  fir GND
MPU6050 SDA    →  rând 20 C  →  fir la același nod SDA (GPIO8)
MPU6050 SCL    →  rând 20 D  →  fir la același nod SCL (GPIO9)
MPU6050 AD0    →  rând 20 E  →  fir GND (adresa 0x68)
```

**Nodul I2C comun (SDA și SCL le împart BMP280 + MPU6050):**
```
GPIO8 (SDA) ──── BMP280 SDA
               └── MPU6050 SDA

GPIO9 (SCL) ──── BMP280 SCL
               └── MPU6050 SCL
```

## Specificații tehnice

| Parametru | Valoare |
|---|---|
| Axe accelerometru | 3 (X, Y, Z) |
| Range accelerometru | ±2g, ±4g, ±8g, ±16g |
| Precizie accelerometru | 16-bit |
| Axe giroscop | 3 (X, Y, Z) |
| Range giroscop | ±250, ±500, ±1000, ±2000 °/s |
| Alimentare | 2.4V – 3.5V (modulul acceptă 5V prin regulator) |
| Interfață | I2C (400 kHz fast mode) |

## Valorile citite în cod

Codul calculează **magnitudinea totală de accelerație** în g:

```
accelG = sqrt(ax² + ay² + az²) / 9.81
```

| Situație | Valoare accelG |
|---|---|
| Senzor plat, nemișcat | ~1.00 g (gravitație) |
| Ținut în mână normal | 0.98 – 1.05 g |
| Mișcare ușoară | 1.05 – 1.20 g |
| Tamper / lovitură | > 1.30 g |
| Aruncat / căzut | > 2.00 g |

## Verificare funcționare

### Pasul 1 — I2C Scan
Trimite `8` în test mode:
```
Found device at 0x68  ← MPU6050
Found device at 0x76  ← BMP280
```

### Pasul 2 — Test senzor
Trimite `6`:
```
Accel: 1.00 g  |  Gyro X: 0.01  Y:-0.00  Z: 0.02 rad/s
```
- Plat pe masă → accel ≈ 1.00 g, gyro ≈ 0
- Înclini → accelG se schimbă, gyro iese din zero

## Calibrare

### Offset giroscop
La repaus absolut, giroscopul poate arăta valori mici nenule (drift).  
Codul curent nu face zero-calibrare. Dacă vrei:

```cpp
// Adaugă în initMPU6050() după setFilterBandwidth:
// Citește 100 de sample-uri și calculează offsetul mediu
// (opțional pentru hackathon)
```

Pentru demo nu e necesar — valorile mici la repaus sunt normale și nu afectează detecția tamper.

### Pragul de tamper în config.h
```c
// config.h — ajustează în funcție de sensibilitatea dorită
// accelG > 1.30 = mișcare semnificativă / tamper
```

Codul în `main.cpp`:
```cpp
if (d.mpuOk && d.accelG > 1.3f) score += 10;  // contribuție risc tamper
```

## Probleme frecvente

| Simptom | Cauză probabilă | Fix |
|---|---|---|
| Nu apare în I2C scan | AD0 neconectat sau VCC lipsă | Conectează AD0 la GND; verifică 3V3 |
| Adresă 0x69 în loc de 0x68 | AD0 la VCC în loc de GND | Mută AD0 la GND sau schimbă `MPU6050_ADDR` |
| accelG mereu 0 | Citire eșuată silențios | Verifică `mpuOk`; rulează I2C scan |
| Conflict cu BMP280 | Adresă accidentală suprapusă | Imposibil — adresele sunt diferite (0x68 vs 0x76) |
| Valori mari la repaus (> 1.5 g) | Vibrații mecanice în jur | Normal în medii cu vibrații; ajustează pragul |

## Note pentru proiect MediTwin

MPU6050 adaugă **dimensiunea de siguranță fizică**:
- Detectează dacă echipamentul a fost atins/mutat (**tamper detection**)
- Poate detecta vibrații anormale în clădire (seism, impact)
- Demonstrează că sistemul nu e doar un monitor pasiv ci un **security node**

**Pentru demo**: lovești ușor masa → accelG crește → contribuie +10 la scorul de risc → sistemul reacționează.

Pin I2C: `PIN_SDA = GPIO8`, `PIN_SCL = GPIO9`  
Adresă: `MPU6050_ADDR = 0x68`
